Database model

De mobiele oplossing voor het probleem van Klabu zal om moeten gaan met verschillende data en het maken van een onderbouwd data model is noodzakelijk voor het gebruik als blauwdruk in de realisatie van de mobile app. In dit onderzoek zal er gekeken worden naar de verschillende deelvragen die geclusterd zijn onder het kader “data technische vraagstukken”.

Uit welke structuur moet het database model bestaan om met de data van de verschillende use cases van de mobile app om te gaan?


Aanpak

Door middel van de onderzoeksstrategie stepping stones zal er aan de hand van de onderzoeksmethode requirement list gekeken worden naar de verschillende use cases waar de app mee om moet kunnen gaan en hieruit wordt een lijst gemaakt van data requirements die als richtlijn gebruikt zal worden bij het ontwerpen van het database model. De volgende deelvragen worden hiermee onderzocht:

- Wat zijn verschillende use cases waar de mobiele app mee moet kunnen omgaan?
- Welke data moet het datamodel kunnen verwerken?

Met behulp van de uitkomsten op deze deelvragen zal er een database model ontworpen worden die geschikt is om deze data te verwerken. Hiermee wordt de volgende deelvraag beantwoord:

- Uit welke structuur moet het datamodel bestaan om met deze data om te gaan?

Zodra deze deelvraag is beantwoord op basis van de voorgaande onderzoeksresultaten zal er door middel van de onderzoeksstrategie workshop met behulp van de onderzoeksmethode prototyping, een prototype gebouwd worden van de mobile app waarin wordt getoetst of het ontworpen database model in de praktijk ook daadwerkelijk effectief om kan gaan met de opgestelde use cases. Resultaten uit deze prototype worden vastgelegd en indien nodig wordt het database model op basis van deze resultaten aangepast.



Resultaat

Om de volledige resultaten van de opgestelde use cases en data requirements te zien, bekijk het database model onderzoeksdocument (zie bronnen).

Aan de hand van de opgestelde use cases en data requirement list is er een database model ontworpen waarin de relaties tussen de verschillende data collecties is vormgegeven. Dit database model is hieronder te zien (klik om te vergroten).






Conclusie

De use cases waar de mobiele app mee om moet kunnen gaan zijn in dit onderzoek in kaart gebracht en op basis hiervan is er een requirement list opgesteld voor welke data verwerkt moet worden door het database model. Met behulp van deze requirement list is er een database model ontworpen waarin de verschillende relaties tussen data collecties vorm worden gegeven. Dit database model zal als blauwdruk gebruikt worden bij het realiseren van de mobiele app, in de loop van de realisatiefase en verdere prototypes kan het database model nog aangepast worden indien nodig.
